# AI 知识库 · 项目愿景 v1.0

## 要做什么

- 每天抓取 GitHub Trending 的 Top 20 仓库
- 仅聚焦 AI 相关仓库，非 AI 仓库不进入知识库
- 用 Agent 对每个仓库做完整分析：总结项目内容、分类打标签、提炼主题、评估价值/热度/可学习性
- 输出结构化知识条目，采用 JSON 作为主格式
- 每条知识条目必须包含：项目名、简介、链接、标签、分析结论、推荐理由

## 不做什么

- 不收录与 AI 无关的仓库
- 不追求全网完整覆盖，只处理 GitHub Trending 的 Top 20
- 不以 Markdown 作为主输出格式
- 不做复杂的人工编辑流程，尽量保持自动化

## 边界 & 验收

- 只要输入源是 GitHub Trending Top 20，且结果能稳定筛出 AI 相关仓库，即符合边界
- 每条入库内容都必须是结构化 JSON，字段齐全且可机器消费
- 每条知识条目必须能清楚回答：这个项目是什么、为什么值得关注、为什么被收录
- 输出内容要能支持后续检索、筛选和二次加工

## 怎么验证

- 随机抽查当天抓取结果，确认数量为 20 条
- 检查是否只保留 AI 相关仓库
- 检查 JSON 结构是否完整，字段是否齐全
- 检查分析结论和推荐理由是否足够具体、可读、可用
